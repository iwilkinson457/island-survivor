# Island Progression - Draft 1

Based on Wilko's current direction.

## Core island rule
Each island should matter because it provides something the player cannot fully replace at home.

Players revisit islands for:
- needed resources
- NPC quests
- progression materials
- better gear components
- clues related to the family search
- boss / high-value opportunities

## Home island
### Main purpose
- home base
- building and crafting
- NPC settlement
- storage
- recovery between runs

### Threat profile
- normally safe from routine enemies
- vulnerable to human raids as base visibility and value increase

### Main progression role
- this is where the player's long-term progress is banked
- the camp grows as NPCs are rescued and housed
- benches, huts, and upgrades are established here

### Revisit reason
- always the centre of progression
- staging area for every run
- place where rescued NPCs unlock new crafting / quests

## Luxury hotel island
### Main purpose
- story origin location
- early major exploration island
- scavenging for civilian supplies

### Main enemies
- zombies

### Resource / loot identity
- food from kitchens and stores
- clothes
- toiletries and daily-use goods
- skincare products
- phones
- laptops
- money
- hotel maintenance items

### Why revisit
- broad civilian loot source
- possible story clues from guest rooms / staff areas / maintenance zones
- useful for food and general supplies
- may contain keys, records, security access clues, or family trail evidence

## Research base island
### Main purpose
- science-heavy progression island

### Main enemies

### Resource / loot identity
- scientific equipment
- chemicals
- lab materials
- outbreak clues
- research notes

### Why revisit

- advanced materials
- high-end medical / chemical / research unlocks
- mutation / outbreak-related discoveries

## Military base island
### Main purpose
- weapon and tactical progression island

### Main enemies

### Resource / loot identity
- guns

### Why revisit

- best combat upgrades
- ammunition supply

## Industrial storage island

## Mining island

### Why revisit

## Boss structure
Each island should have a boss that has a chance to spawn.

### Design rule
Bosses should reflect the fantasy of the island, not feel random.

Examples:
- hotel island = elite infected guest / security brute / infected manager / kitchen horror
- research island = mutant experimental subject / escaped test organism
- military island = heavily armed commander / armoured unit leader
- industrial island = massive infected worker / militia foreman / power-armour-style brute if the setting supports it
- mining island = tunnel brute / infected excavation giant / gang-controlled extraction boss

### Recommended boss role in progression
Bosses should provide one or more of:
- rare crafting material
- recipe unlock item
- NPC quest item
- keycard / map fragment / family clue
- high-value extraction target

## Progression logic between islands
Current structure implies a chain like this:
1. Home island survival
2. Rescue mining engineer
3. Unlock mining island
4. Use materials to improve tools / camp / capability
5. Reach hotel / industrial / military / research islands depending on unlock structure
6. Rescue more specialists and expand the camp
7. Follow family clues deeper into the island network

## Recommended progression rule
Do not make islands strictly one-and-done.
Instead:
- each island should have an initial unlock phase
- then a repeatable resource / objective phase
- then quest-driven revisit reasons later

This will make the world feel more like a connected loop than a series of levels.

## Strong design principles for islands
1. Every island needs a strong identity.
2. Every island needs a reason to revisit.
3. Every island needs at least one exclusive progression value.
4. Enemy type should match island fantasy.
5. The island should support both quick runs and deeper riskier runs.
