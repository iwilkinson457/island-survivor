# Extraction: Dead Isles — Mining Island Multi-Run Progression

## purpose
This document defines how Mining Island should unfold across multiple runs rather than being cleared in one visit. The island should teach that expedition progress is staged, that some routes stay locked until the player returns home, and that island mastery happens over repeated raids.

## core design rule
Mining Island must not be designed as a one-and-done clear.

The player should understand that:
- the first run is partial
- key areas are inaccessible at first
- returning home matters
- specialist interpretation and support matter
- extraction before full completion is part of the intended loop

## progression philosophy
Mining Island should teach one of the game's most important rules:

**you do not finish an island in one raid**

Instead, the player should:
1. scout and survive
2. recover partial progress
3. extract
4. return home
5. consult the relevant NPC
6. unlock the next layer
7. go back in

This is one of the clearest ways to reinforce the extraction-survival structure.

## Mining Island structure overview
Mining Island should be split into progression layers:

### layer 1 — outer camp and living area
- landing zone
- worker living quarters
- mess hall
- office or admin spaces
- storage sheds and outer utility spaces

### layer 2 — locked mine access
- main gate or secured route into the mine area
- visible but inaccessible first-run objective barrier

### layer 3 — mine interior progression
- coal route
- iron route
- deeper industrial spaces
- locked or dangerous sections
- boss zone

This makes the island readable and staged.

## first run design goal
The first run to Mining Island should only reach the camp living area.

The player should not enter the actual mine on the first run.

### reason
This enforces the idea that:
- progress is partial
- important routes are locked
- clues matter
- the engineer matters
- the player is expected to leave and return later

## first run objectives
The first run should focus on:
- reaching the worker camp safely
- exploring living and support spaces
- finding useful supplies
- discovering the mine is locked
- recovering documents or records related to the mine
- extracting back to Home Island

## locked mine gate rule
The route into the mine should be clearly locked on the first visit.

### design purpose
This should communicate:
- the island is not fully solvable yet
- there is a real reason to return home
- this is not a fake invisible wall, but a progression lock

### presentation options
The gate can be represented by:
- a proper locked industrial gate
- secured access door
- fenced checkpoint
- collapsed access point with documented access requirement

The cleanest first-pass approach is a locked gate requiring key or authorised access.

## first-run key discovery
The player should find documents in the worker camp or admin area that explain or imply:
- the mine route exists
- access is restricted or keyed
- the engineer will understand the meaning or know how to use the information

### examples of good document content
- mine access paperwork
- shift or security records
- route diagrams
- keycard reference note
- operations memo

## return-home requirement
After the first Mining Island run, the player must return home with the recovered documents.

### engineer role
The engineer should review the documents and then:
- explain what the access requirement means
- provide the key or means of access
- unlock the next Mining Island objective

This is a very strong progression moment because it proves the specialist is useful in a practical way.

## first run emotional and gameplay identity
The first Mining Island run should feel like:
- a tense scouting mission
- profitable, but incomplete
- revealing, but not resolving the island
- a successful expedition even though the mine stays shut

That feeling is important. The player should not think they failed because they did not reach everything.

## first run likely rewards
The first run should mainly reward:
- food and camp supplies
- scrap and basic industrial salvage
- notes and logs
- the key documents needed for engineer interpretation
- a stronger understanding of the island layout

It should not yet pay out:
- full ore progression
- mine access rewards
- boss rewards
- full industrial extraction value

## second run purpose
After the engineer processes the documents, the second run should unlock actual mine access.

### second run focus
- re-enter Mining Island with key access
- open the locked route
- begin coal and/or initial mine progression
- push deeper than before
- extract with early mining rewards

## third and later run purpose
Later runs should continue the island arc by escalating depth and value.

### later run goals can include
- deeper coal progression
- iron route access
- stronger clue discovery
- boss approach and kill
- more valuable industrial materials
- unlocks that point toward the next island

## run-by-run structure recommendation
### run 1 — scout and recover documents
Reach the worker camp and living area only.
Find the documents.
Discover the mine is locked.
Extract.

### run 2 — access the mine
Return after the engineer interprets documents and provides the key or access solution.
Push into early mine spaces.
Secure first meaningful mining rewards.
Extract.

### run 3 — deeper progression
Push further into coal or iron areas.
Handle stronger resistance.
Recover stronger materials and clues.

### run 4 or later — boss and major payoff
Reach the deeper milestone zone.
Defeat the boss.
Claim major rewards and next progression hook.

These run counts can compress or stretch in tuning, but the structure should remain staged.

## level design implication
Mining Island should visually support this staged understanding.

### the player should be able to see or infer
- there is more beyond the first-run area
- the locked gate matters
- the mine is a future target
- the camp and mine are distinct layers of the island

This helps the player accept leaving and returning.

## loot structure by run
### run 1 loot identity
- worker camp food
- scrap
- logs and notes
- practical expedition supplies
- key documents

### run 2 loot identity
- first mine resources
- coal and industrial progression items
- stronger route knowledge

### later run loot identity
- iron ore
- deeper industrial materials
- dangerous clues
- boss rewards
- stronger progression items

## quest and progression integration
Mining Island should tie directly into the engineer progression chain.

### clean chain example
1. rescue engineer
2. build and support engineer
3. unlock Mining Island travel
4. first Mining Island run reaches camp only
5. recover mine documents
6. return documents to engineer
7. engineer gives key or access solution
8. second run opens mine access
9. later runs deepen mine progression

This is a very strong loop.

## player messaging goal
The game should clearly teach that leaving an island unfinished is not failure.

### intended message
A good extraction can be:
- surviving
- learning the place
- recovering a clue
- unlocking the next step

not only clearing every enemy or reaching every room.

## avoid
- designing Mining Island so the player can brute-force the whole thing in one early trip
- locking the island with no understandable reason
- making the first run feel pointless
- making the documents feel like arbitrary key items instead of meaningful site information
- making the player think they missed content by extracting early

## summary
Mining Island should be the game's clearest early lesson in staged expedition progression.

### core intended flow
- first run: camp only, mine locked, documents recovered
- return home: engineer interprets and unlocks access
- second run: first mine entry
- later runs: deeper progression, better rewards, boss, and next hooks

This structure strongly reinforces the extraction-survival identity of the game.
