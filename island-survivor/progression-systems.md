# Progression Systems - Draft 1

Based on Wilko's current direction.

## Core progression philosophy
This game should not be about clearing one island and never returning.
Instead, progression should work like this:
- unlock access to an island
- extract what you can survive to bring home
- use new resources and NPC unlocks to improve your capability
- return to earlier islands when you need materials, gear, food, quest items, or replacement equipment

The world should feel like a connected survival network, not a sequence of disposable levels.

## Extraction rule
The extraction rule remains central:
- if you die on an island, you lose what you were carrying
- you respawn at home
- progress only banks when you get back alive

This makes every run meaningful.

## Island completion rule
An island is never truly complete.
However, the player can reach functional milestones on an island.

### An island is considered progression-complete when the player has:
- found what unlocks the next island or major progression step
- rescued any key NPC associated with that island

### But the island remains relevant because:
- resources are still needed
- better loot may still be desirable
- quests may send the player back
- replacement gear may be required after death
- boss spawns / high-value encounters can still happen

## Island unlocking system
Current preferred system:
- NPC quests
- maps
- clues

This should be the main route to expansion.

### Recommended support unlocks
These can sit underneath the main system:
- fuel requirements
- boat repair / range improvement
- access codes / keys
- specialist guidance from rescued NPCs

That way progression feels layered rather than binary.

## Home base progression system
### NPC hut unlock rule
When an NPC is rescued:
- the player can build that NPC's hut
- once the hut exists, the NPC becomes available at the base
- crafting or services may still be partly gated behind quests

This is a strong system because rescue alone is not enough. The player must invest in the camp to convert rescue into permanent utility.

## Base building scope
Target scope: medium depth.

### Confirmed wanted base systems
- huts for NPCs
- player house / shelter
- storage crates
- crafting benches
- walls / gates
- traps
- watchtower
- campfire / cooking
- farm plots
- generator / solar power
- water collection
- lighting

## Player house direction
The player house should be:
- freeform but simple
- one level only initially
- built from floor, wall, roof, window, and door pieces
- initially wood-based
- later upgradable to stone via the builder

This is a very good scope choice. It gives ownership and creativity without forcing a huge survival-builder system from day one.

## Raid system
### Current desired raid structure
Raids should be event-based and should:
- target the home base
- scale with camp visibility / growth
- involve human raiders, not zombies
- damage or destroy structures if the player fails
- steal resources if the player fails
- be mitigated by traps and defences

### Why this works
This creates a strong strategic loop:
- building more helps you survive and progress
- building more also increases the attention your camp attracts
- the player is constantly balancing growth, risk, and preparedness

## Zombie kill incentive system
The player wants a reason to kill zombies, but does not want tedious corpse-by-corpse looting.
That is the right instinct.

### Best-fit reward model
Use an automatic or lightweight pickup system where zombies contribute to a service-crafting economy.

### Recommended rule
Zombies and special infected should contribute materials or proof tokens automatically or near-automatically when killed, without requiring manual body search every time.

### Reward use
These rewards should be spent on:
- NPC crafting services

This is especially strong because it means:
- finding the ingredients is not enough on its own
- killing zombies also contributes to your right to request crafted items
- combat and progression stay linked

### Suggested implementation directions
Possible flavours:
- outbreak residue
- infected tags
- contaminated samples
- bounty marks
- salvage markers

The exact fiction can be chosen later, but the key rule is already clear:
- killing enemies should support the service economy
- collecting rewards should not become repetitive busywork

## Crafting progression structure
### Tier 1 - field crafting
Craft from inventory:
- spear
- axe
- pickaxe
- simple primitive items

### Tier 2 - bench crafting
Requires benches or stations:
- better tools
- refined materials
- more advanced survival gear
- some weapon progression

### Tier 3 - NPC services / specialist crafting
Requires:
- rescued NPC
- NPC hut
- needed ingredients
- enough zombie reward / service currency if applicable
- sometimes quest completion

This creates a nice progression stack:
- survive alone first
- build camp second
- unlock specialists third
- pursue higher-end gear through island loops

## Loot system
Current preferred direction:
- island structure is fixed / handcrafted
- loot is randomised from a list appropriate to that island

### Important loot rule
Island loot should stay thematically correct.
Example:
- you do not find an M16 in the hotel

This is a very strong rule and should stay explicit in the design.

### Recommended loot model
Each island should have:
- fixed high-value landmark loot possibilities
- semi-random general loot tables
- enemy-specific drops
- occasional rare finds
- optional boss-linked rewards

This preserves replayability while keeping the world believable.

## Enemy randomisation
Recommended direction based on current agreement:
- fixed island layouts
- partly random loot
- random roaming enemies
- random boss chance
- random raider presence

This gives tension and replayability without making islands feel procedural and meaningless.

## Hotel role in progression economy
The hotel should be an important revisitable island because it provides:
- food
- civilian supply items
- broad scavenging opportunities
- possible story clues

This makes it a good mid-loop support island rather than just a one-time story level.

## Strong progression principles now locked in
1. Extraction success is what banks progress.
2. NPC rescue + hut building turns survivors into permanent systems.
3. Base growth creates raid risk.
4. Zombie kills should fuel crafting services without corpse-loot tedium.
5. Islands remain useful over time rather than being one-off stages.
6. Loot must remain appropriate to the island fantasy.

## Open design questions still remaining
- What exact name / fiction should the zombie reward currency use?
- How frequently should raids occur, and what triggers the first one?
- How many build pieces does the first playable really need?
- Which NPC services require only ingredients versus ingredients + zombie reward currency?
- How should boss spawn chance work without feeling random and frustrating?
