# Extraction: Dead Isles — Mining Island First Visit Mission Flow

## purpose
This document defines the intended first visit to Mining Island. The first visit should be a partial expedition focused on the worker camp and living area only. The mine itself should remain locked until the player returns home with recovered documents and the engineer provides access.

## first-visit principle
The first Mining Island visit must teach that islands are not always cleared in one raid.

The player should succeed by:
- reaching the island
- exploring the camp living area
- finding useful supplies
- discovering the mine is locked
- recovering documents
- extracting safely

That should count as a successful run.

## first-visit allowed space
The first visit should mainly cover:
- landing zone
- worker living area
- mess hall
- admin or office space
- storage sheds
- outer utility spaces

The player should not enter the actual mine on this visit.

## locked mine rule
A gate, secured route, or access barrier should prevent entry into the mine on the first run.

### intended effect
The player should immediately understand:
- there is more here
- I cannot do it all right now
- I need to take something useful home first

## first-visit objectives
### practical objectives
- survive the journey in
- scout the worker camp
- loot supplies
- locate the locked mine access
- search the camp and office spaces
- recover key documents
- extract back to Home Island

## document role
The documents found on the first visit should be the bridge between Mining Island and the engineer.

### document purpose
They should tell the player or imply that:
- the mine requires controlled access
- the camp had an access process or key system
- the engineer will understand what this means

### examples
- access memo
- mine operations paperwork
- route diagram
- shift office document set
- keycard or access reference file

## first-visit emotional identity
The first run should feel:
- tense
- exploratory
- incomplete in a good way
- profitable enough to matter
- clearly leading to the next step

## recommended flow
### 1. arrival and landing
Player reaches Mining Island and enters the outer camp zone.

### 2. camp approach
Player clears or avoids early threats and gets into the living area.

### 3. camp scavenging
Player loots food, scrap, notes, and practical supplies.

### 4. gate discovery
Player reaches or sees the locked mine access and understands the route is blocked.

### 5. office/admin search
Player searches for explanation and finds the required documents.

### 6. extraction decision
Player recognises the run is complete enough to leave.
Returns to the boat and extracts.

## reward identity for first visit
The first Mining Island visit should mainly reward:
- food
- scrap
- worker notes or logs
- practical salvage
- mine access documents

It should not yet reward:
- major ore extraction
- coal or iron progression in full
- boss rewards
- full island completion

## return-home follow-up
After the first visit, the player should return to the engineer with the documents.

### engineer result
The engineer should:
- explain the meaning of the documents
- identify what is needed to open the mine route
- provide the access key or unlock solution
- set up the next Mining Island objective

## design rule
The first visit should never feel like wasted time just because the mine stays closed.

It should feel like a meaningful scouting and clue-recovery run.

## avoid
- allowing easy mine entry on the first trip
- making the locked gate feel arbitrary
- making the documents too hidden or too random
- making the first visit overly long
- making the player think they failed by extracting before the mine opens

## summary
The first Mining Island mission flow should be:
1. arrive
2. scout camp
3. loot camp
4. find locked mine gate
5. recover documents
6. extract
7. return to engineer
8. unlock second-run mine access
